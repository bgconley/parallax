# Semantic Chip Legend

The chip system is intentionally constrained. Do not allow chip explosion.

## Blue — Informational / focus / planning certainty
Use for:
- Task
- Needs timing
- Time-sensitive
- Protected
- Best next step
- Running now

Meaning:
This item has structure, immediacy, or focus relevance.

## Green — Adaptive / supportive / successful / safe
Use for:
- Done
- Can move
- Can shrink
- Adjusted for energy
- Adaptive
- Personal model
- Protected essentials

Meaning:
The system is helping, softening, or affirming viability.

## Purple — Process / follow-up / insight / internal effort
Use for:
- Follow-up
- Later
- Low-friction
- Often sticky
- Still learning
- Active + elapsed
- Needs focus

Meaning:
This item relates to cognition, reflection, process, or ongoing learning.

## Gold — Flexible / caution / optional warmth / ideas
Use for:
- Idea
- Flexible
- Shortened
- Low energy
- Light caution

Meaning:
This item is optional, exploratory, or condition-sensitive.

## Neutral — Unclassified / passive / structural
Use for:
- Saved
- Optional
- Not for today
- Routine

Meaning:
The item has no urgent behavioral interpretation.

## Chip density rules
- A list row should generally show **1 primary chip**.
- Expanded or more detailed rows may show **up to 2 chips**.
- 3 chips is allowed only for special views like timing review or recovery, where the chips convey distinct value.
- If a row needs more than 2 chips to make sense, the screen should probably be a detail view or card.
