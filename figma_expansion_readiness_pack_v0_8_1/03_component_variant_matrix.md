# Component Variant Matrix

## 1. Page header
Required parts:
- circular nav button left (menu or back)
- centered title
- optional supportive subtitle
- circular contextual action right (sparkle / assistant)

Variants:
- Root header: menu left
- Detail header: back left
- Sheet header: handle + centered title, no large nav chrome

## 2. Helper AI card
Purpose:
- reassure
- explain what the system is doing
- reduce pressure
- contextualize adjustments

Required parts:
- soft icon circle left
- 1–2 lines of primary message
- optional supporting line
- optional small meta line
- optional heart icon right

Never use for:
- long paragraphs
- tutorials
- error dumps
- generic motivational filler

## 3. Summary card
Used at the top of a surface to represent the most important object or action.
Examples:
- current focus
- task summary
- routine summary
- timing session summary

## 4. List rows
Core list row anatomy:
- leading icon or numbered badge
- primary label
- optional secondary label
- optional 1–2 chips
- trailing chevron

Variants:
- task row
- inbox row
- routine step row
- timing step row
- settings row

## 5. Metric tile
Used only in review, timing, or insight surfaces.
Should remain sparse and calm.
No noisy dashboards.

## 6. Footer action bar
Three-action layout is canonical.
Order:
- left: primary
- middle: secondary
- right: tertiary

If only two actions exist:
- left primary, right secondary

If one action exists:
- full-width primary
