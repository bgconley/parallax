# Navigation and Layout Contract

## Root screens
Examples:
- Today
- Inbox
- Weekly review
- Settings

Rules:
- menu button left
- sparkle/context button right
- title centered
- subtitle optional but preferred when it meaningfully orients the user
- first major card appears near the top
- main list/content scrolls below
- bottom actions anchored when important

## Detail screens
Examples:
- Break it down
- Task details
- Routine builder
- Timing review

Rules:
- back button left
- sparkle/context button right when contextual AI help exists
- stacked cards dominate
- avoid full-page forms
- action bar remains anchored at bottom

## Bottom sheets
Examples:
- Calibrate timing
- quick choose actions
- save / route decisions

Rules:
- visible handle
- centered title
- 1-line supportive subtitle
- list of options with simple radio or tap selection
- 2-button footer whenever possible

## Timing flow
Must feel slightly denser than standard views, but still humane.
The timer is the focus object. Step list is secondary. Friction tags and review aids come later.

## Review / insight surfaces
Use cards and sections, not noisy dashboards. Charts should be supportive and subdued.
