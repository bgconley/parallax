# Figma Usage Playbook

## Recommended workflow for the Figma-connected agent

### Step 1 — Load authority materials
Always load:
- anchors
- source docs
- system boards
- tokens / inventory

### Step 2 — Pick nearest analogs
For each new screen, choose 2–4 nearest existing screens.

Examples:
- New inbox variant → Today + Quick Capture + Inbox continuation + state matrix
- New routine screen → Routine Builder + Reset + Timing Session
- New review screen → Weekly Review + Timing Review + State Matrix

### Step 3 — Extend using the same grammar
- same title treatment
- same card radius family
- same bottom action hierarchy
- same semantic chip system
- same calm helper-card tone

### Step 4 — Stress test
Each new frame should also get variants for:
- empty
- loading
- offline or sync-pending
- long text
- large dynamic type

### Step 5 — Handoff discipline
For each new frame, document:
- primary question answered by the screen
- primary CTA
- optional CTAs
- components used
- state variants required
- any new component introduced

No new component should be introduced unless the existing library truly cannot express the interaction.
