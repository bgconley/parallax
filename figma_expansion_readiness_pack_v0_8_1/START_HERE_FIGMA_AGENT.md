# Start Here — Figma Agent Continuation Contract

## Objective
Continue fleshing out the app in Figma while preserving the exact product posture already established:

- calm on entry
- low-friction capture
- progressive structure only when useful
- list-first, not dashboard-first
- AI as contextual assistance, not a chat-centric experience
- humane, non-shaming recovery language
- temporal learning as supportive realism, not surveillance

## Authority order
When references conflict, use this order:

1. `anchors/` approved screens
2. `source_docs/` markdown source of truth
3. `tokens_inventory/design_tokens.json`
4. `system_boards/` visual grammar boards
5. `continuation_screens/` sibling examples

## Do not invent
- a new navigation grammar
- a new semantic color family
- a chatbot-first shell
- dashboard-heavy home screens
- dense admin forms for ordinary task work
- extra chip categories unless truly necessary

## Safe to extrapolate next
- inbox variants
- task detail variants
- routine run / routine edit
- weekly review / timing insights
- settings / personalization
- empty, loading, offline, sync-pending, long-text, error states
- privacy / consent and export-delete surfaces, using the same card grammar

## Each new screen must answer one primary question
Examples:
- Inbox: what did I capture and what should I do with it?
- Task details: how should this task fit real life?
- Reset: how do I continue after today slipped?
- Timing review: what actually happened and what should the model learn?

## Per-screen rules
- One clear primary action.
- Supporting actions may exist, but should visually step down.
- AI explanation cards should be calm and brief.
- Lists and cards are the dominant containers.
- Sheets are preferred over full-screen form flows when the interaction is short or reversible.
- Chips communicate meaning; they do not replace primary content.
